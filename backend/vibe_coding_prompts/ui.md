# 设计一套为这个项目准备的灵动交互系统

### 功能简述

[重要]1.简述：后端的AGENT可以编写一段HTML代码，在前端虚拟形象旁边显示，其中可以包含有模型自定义的UI交互元素（如复选框，文本框等），用户可以选择操作后，通过API将选择的操作传给后端，称之为灵活窗口

[重要]2.目的：在人机交互过程中，如遇到类似于如安装软件这样的复杂任务，由于本项目基于语音进行人机交互，显然需要模型大量与用户确认需求，但通过语音等方式，不但效率极低，不够明确，且UE较差，此时，便可以通过这个功能与用户进行更加直观的交流

### 技术实现

1.你必须为后端AGENT添加适配与这些功能的TOOLS,同时在docstring告诉模型有关这个功能的详细信息

2.在trigger_manager中添加以事件（Event）触发的触发器，以供该功能使用，如果有必要，修改对应的处理trigger_manger事件触发的backend-main.py程序
3.在后端中添加用于前端”灵动交互“窗口与后端通信的API，功能包括但不限于 唤醒模型，传输结果信息等，并且使用 （2）中提到的新触发器触发对话

3.1 特别说明：”灵动交互“窗口与后端通信，唤醒模型等的API调用权完全归属于AGENT，并且要为其封装好Javascript语言的调用API函数，可以使用我已有的API端口，并进行修改

4.为前端编写这个功能的前端部分，即对后端HTML代码的展示，在虚拟形象边的一个单独窗口实现

5.”灵动交互“窗口唤起后，对于后端是”非阻塞式“的，即到了需要的时候才会调用API唤起模型，进入下一步操作，否则任然可以通过语音识别，其他类型触发器触发等方式进行对话

5.1 附加说明:这也就是说，你需要保证某一个后端AGENT对于 ”灵动交互“ 相关功能的TOOL CALL 必须是非阻塞式的，否则会导致在”灵动交互“ 相关功能进行时，如果其他对话触发，调用OPENAI 对话API时 出现 TOOL CALL 和 已完成的工具返回值 不相符的错误 
6.请在后端faust_backend新建一个nimble.py，可以在这个文件中实现逻辑
